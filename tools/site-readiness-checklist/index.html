Site-Readiness Checklist

What a plant needs in place before a one-day vision-grading install.

Singh Automation · Version 1.0 · 2026-04-27

How to use this document

This is the same checklist a Singh Automation field engineer walks during a pre-install site survey. It is built around one promise: ceiling-mounted vision grading installs over a single weekend, with no line modification and no PLC re-sequencing. Everything in this document exists to make that promise true on your plant.

The checklist has six sections: ceiling and overhead access, network and IT, electrical and power, PLC and control-system integration, lighting and ambient conditions, and operations and people. Each section has two parts — a walk-the-floor checklist (yes/no items the QA, ops, or maintenance lead can answer in person) and a what we measure on survey day (the spec we capture, used to size the install).

If you fill this out before we arrive on site, the site survey becomes a 90-minute confirmation walk instead of a half-day of investigation. That alone has saved real installs from sliding to a second weekend.

Section 1 — Ceiling and overhead access

This is the section that determines whether the install is one weekend or two. Read it first.

Walk-the-floor checklist

  • The ceiling directly above the grading station is unobstructed by ductwork, sprinkler heads, light fixtures, or catwalks for at least a 6-foot diameter footprint
  • The clear ceiling height above the shackle line at the grading station is at least 11 feet
  • Existing overhead structure (roof joist, beam, or rated ceiling) can support a 60 lb mounting frame
  • There is a clear path for a two-person crew with a 10-ft step ladder or rolling scaffold to reach the mount point without having to stop the line
  • No active steam, hot-water spray, or aerosolized water pattern intersects the planned mount footprint at production conditions

What we measure on survey day

  • Ceiling height directly above the grading station, in inches
  • Distance from mount centerline to nearest obstruction (light, duct, sprinkler) in all four cardinal directions
  • Photograph of the ceiling directly above the grading station, taken from line height looking up
  • Beam or joist spacing in the vicinity of the mount point
  • Ambient temperature and humidity at the mount height under normal production conditions (this drives the camera-housing rating)
If this section is a problem. If the ceiling is obstructed in a way that can't be worked around — usually a duct run or fixed catwalk directly above the grading station — the most common solution is a side-mount frame with the cameras on a horizontal arm rather than directly overhead. That adds about half a day to the install and shifts go-live from Sunday afternoon to Sunday evening. It does not push the install to a second weekend. What does push to a second weekend: discovering the obstruction during install instead of survey. That is the entire reason this section exists.

Section 2 — Network and IT

The vision system needs one wired ethernet drop within reach of the mount frame. That is the entirety of the network footprint.

Walk-the-floor checklist

  • There is a managed ethernet drop within 100 feet of the planned mount point
  • The drop sits on a network segment that has line-of-sight to the PLC's ethernet network (same VLAN, or a configured route)
  • IT has been told that a new device with a static IP will be added to that segment, and a static IP has been pre-allocated
  • The plant network has at least 100 Mbps available between the new drop and the PLC for the duration of the install (the system's bandwidth requirement is much lower in steady state, but calibration uses more)
  • Plant IT has confirmed there is no MAC-address allowlist that would block a new device, or has the process to add the device's MAC

What we measure on survey day

  • Cable distance from nearest available drop to the planned mount point (informs whether we need a cable pull or can splice in)
  • Network segment / VLAN of the available drop, and whether it routes to the PLC
  • Round-trip latency from the planned mount point to the PLC over the network (must be under 5 ms)
  • Whether the plant uses an air-gapped OT network, and if so, how the OT network is bridged for vendor diagnostics
If this section is a problem. If the nearest drop is on the wrong side of the plant, the install gains a half-day for cable pull. If IT is not aware the install is happening — the most common version of this problem — the install can stall mid-day Saturday waiting for a static IP. The single best preventive action is to have plant IT on the survey call when the engineer arrives. Five minutes on the survey saves an entire afternoon.

Section 3 — Electrical and power

The vision system pulls under 200 watts at steady state. Power is rarely the bottleneck. Documentation is.

Walk-the-floor checklist

  • There is a 120 VAC, 20 A circuit available within 50 feet of the mount point, with at least one open outlet on that circuit
  • Or: maintenance has confirmed they can pull a new 20 A circuit to a junction at the mount point during the install window (typically a 2-hour electrician task)
  • The circuit is on a clean side of the plant electrical (i.e., not sharing a panel with high-current loads like compressors that cause voltage sag)
  • The circuit is GFCI-protected if any portion of the run passes through a wash-down zone

What we measure on survey day

  • Outlet location and panel-of-origin, photographed
  • Voltage at the outlet under load (we plug a small load and meter it)
  • Whether the circuit is shared with any line equipment (we want it isolated from line-control circuits — the system is fault-tolerant either way, but isolation is cleaner)
  • Cable run path from outlet to mount point
If this section is a problem. Almost never an install blocker. The most common issue is that a circuit exists but isn't documented, and finding the right breaker takes longer than the install of the cable run itself. If there is no usable circuit within reach, your maintenance team can pull one in parallel with the rigging step on Saturday morning — no schedule impact.

Section 4 — PLC and control-system integration

This is the section the QA and controls teams care about. The good news: the system asks very little of your PLC.

Walk-the-floor checklist

  • The PLC controlling the grading station is identified by manufacturer, model, and firmware version
  • The PLC supports either EtherNet/IP or Modbus TCP/IP (every major brand from Allen-Bradley to Siemens to Mitsubishi does)
  • The PLC has at least two free integer tags available for the vision system to write to (grade class and confidence score)
  • The controls engineer responsible for that PLC is available for a 1-hour handshake test on Saturday afternoon during install
  • The plant has change-management approval for adding the vision system as a new ethernet device on the OT network

What we measure on survey day

  • PLC make, model, firmware
  • Network protocol (EtherNet/IP, Modbus TCP, etc.) and IP address of the PLC
  • Two tag names confirmed by the controls engineer that the vision system will write to
  • Existing control logic at the grading station (we want to read it, not change it — but we want to read it)
  • Whether the plant has a HMI display we can optionally publish status to, or whether status stays in the vision system's own dashboard
If this section is a problem. Three legitimate scenarios: (1) the PLC is end-of-life and doesn't support modern ethernet — rare on mid-tier plants; if it shows up, we can run a serial gateway, but the install adds a day. (2) The controls engineer is not available on Saturday — calendar problem, not technical; resolve at survey. (3) The change-management process for adding an OT-network device takes weeks — the single most common non-engineering delay. Start the change-management ticket at survey, with the install date on it. The technical work is fast; the paperwork has to start in parallel.

Section 5 — Lighting and ambient conditions

Vision systems are sensitive to lighting changes. Most poultry plants are already lit consistently for human graders, which is most of the work.

Walk-the-floor checklist

  • Existing overhead lighting at the grading station is consistent across the line and does not flicker (modern LED arrays and most fluorescent fixtures are fine)
  • No direct sunlight reaches the grading station from windows or skylights at any time of day
  • No moving shadows fall across the grading station from operators, ladders, or transient equipment during a normal shift
  • Existing wash-down spray patterns do not put visible water mist into the camera viewing zone during production
  • The grader's view of the bird is unobstructed (this is also what the vision system needs — same geometry)

What we measure on survey day

  • Lux at the bird at line height, taken at three points along the grading station and at three times during the shift (start of shift, mid-shift, end of shift)
  • Whether the existing lighting is on a circuit that gets switched off during clean-down (and whether the vision system's lighting needs the same switching)
  • Color temperature of the ambient lighting, photographed with a color reference card
If this section is a problem. The vision system can supply its own dedicated lighting, mounted in the same overhead frame as the cameras. This adds one fixture to the rigging step on Saturday morning. It does not affect the schedule. The reason we still survey ambient is that the system performs better — and calibrates faster — when the ambient and dedicated lighting cooperate rather than fight.

Section 6 — Operations and people

The technical install is the easy part. Most weekend installs that miss are missed because of the human side of the project.

Walk-the-floor checklist

  • The plant manager has approved the weekend install window in writing
  • The QA director is the named owner of the shadow-mode sign-off on Sunday morning
  • At least two of the plant's regular graders are scheduled for the Sunday production shift, so they're present for the live cutover
  • Maintenance has a 2-hour standing window on Saturday morning for the rigging step (or has confirmed Singh Automation is rigging on its own)
  • The plant's regular Sunday production schedule has been communicated to Singh Automation in advance, so the shadow shift slots into normal production rather than displacing it
  • Someone — typically the QA director or the controls engineer — has been designated as the install-day point of contact and has Singh Automation's mobile number

What we measure on survey day

  • Names and roles of the people listed above, with phone numbers
  • The exact production schedule for the install weekend, hour by hour
  • Any planned plant events (audits, customer visits, USDA inspector schedule changes) that could intersect the install
  • The plant's preferred channel for status updates during install (text thread, email, walkie-talkie)
If this section is a problem. This is the section where most weekend installs that fail, fail. The pattern is always the same: the technical scope was clear, but the QA director wasn't in the room when the install was scheduled, and they're either not on site for the shadow shift or not empowered to sign off. Resolve at survey. The single best question to ask at survey is: “Who signs off on shadow-mode on Sunday morning, and are they confirmed to be on site?”

The three things that actually slow installs

If you have read nothing else in this document, read this. Across the installs we have done, three issues account for the vast majority of all install delays:

1. Ceiling obstruction discovered at install instead of survey. A duct run or a sprinkler head that no one mentioned because no one was looking up. The fix is the side-mount frame; the prevention is a thorough Section 1 walk.

2. Network drop is on the wrong side of the plant. A half-day cable pull added to a one-day install. The fix is fast; the prevention is a thorough Section 2 walk with plant IT in the room.

3. Change-management paperwork for adding a new OT device takes weeks instead of days. The technical install is fast; the IT/security paperwork has to start in parallel. The fix is administrative; the prevention is starting the change ticket at survey time, with the install date on it.

Note that none of the three is a control-system problem, a calibration problem, or a software problem. The technology works. The schedule risk lives in the building and in the calendar.
